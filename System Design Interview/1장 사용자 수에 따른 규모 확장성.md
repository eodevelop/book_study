1장. 사용자 수에 따른 규모 확장성
===============================

수직적 확장(Vertical scaling) 의 단점
-----------------
1. 수직적 확장에는 한계가 있다. CPU나 메모리 등을 무한대로 증설 할 방법은 없다.   
2. 수직적 규모 확장법에는 자동 복구 방안이나 다중화 방안을 제시하지 않는다.
##### 여러개의 서버가 있어야 하나의 서버가 문제가 생겼을때 다른 서버로 대체하는 등의 처리가 가능하다는 의미?
<br/>

로드 밸런서
----------
사용자는 로드 밸런서의 공개 IP 주소로 접속 후 로드밸런서에 의해서 서버로 연결된다.   
즉 웹 서버는 클라이언트의 접속을 직접 처리하지 않는다.

그리고 더 나은 보안을 위해서 서버간 통신에서는 내부 IP 주소를 사용한다.  
로드밸런서는 웹 서버와 통신하기 위해서 이 내부 IP를 사용한다.
